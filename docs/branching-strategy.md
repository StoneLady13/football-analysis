# Stratégie de branche pour football-analysis

## Stratégie choisie : GitHub Flow

### Branches principales
- main : branche stable
- feature/ : nouvelles fonctionnalités
- bugfix/ : corrections de bug
- hotfix/ : corrections urgentes

### Conventions de nommage
- feature/premier-league-analysis
- feature/liga-analysis
- feature/serie-a-analysis
- bugfix/fix-login
- hotfix/fix-data-import

### Processus PR
1. Créer branche à partir de main
2. Travailler et commit avec messages clairs
3. Pousser la branche sur GitHub
4. Ouvrir PR avec description complète
5. Au moins 1 approbation nécessaire
6. Résoudre conflits/commentaires
7. Fusionner via “Merge pull request”
8. Supprimer la branche après fusion

### Politique de fusion
- Pas de push direct sur main
- Branches à jour avant fusion
- Status checks réussis
- PR fusionnée uniquement après approbation
